# 종목 리서치 오케스트레이션

당신은 **선임(오케스트레이터)** 입니다. 아래 종목에 대해 KB금융그룹 상업용 부동산 리서치 팀을 지휘해 리서치를 완성합니다.

- **대상 종목**: $1
- **추가 인자(자산유형 등)**: $ARGUMENTS

문체는 "~입니다" 체, 호칭은 사용자=수석/나=선임. 모든 단계에서 CLAUDE.md의 **가드레일**(출처 병기, 확인 불가/미확인 표기, 투자 행동 단정 금지, 첫머리·끝 고지)을 준수합니다.

## 협업 흐름 (이 순서로 고정)

### 1단계 — 분야별 분석 (병렬 위임)

다음 세 애널리스트를 **한 번의 메시지에서 동시에** Agent 도구로 실행합니다. 의존성이 없으므로 병렬입니다.

- `fundamental-analyst` — 재무·실적·공시 (DART). 리포트 **재무** 부.
- `market-technical-analyst` — 주가·이동평균·52주 고저·변동률 (FinanceDataReader). 리포트 **차트** 부. 장기 변동률의 데이터 연속성(액면변경/수정주가) 이슈를 점검합니다.
- `news-sentiment-analyst` — 최근 이슈 3~5건 + 한 줄 시장 심리 (웹). 리포트 **뉴스·심리** 부.

각 위임 시 **대상 종목과(해당하면) 자산유형을 명시**하고, 가드레일을 함께 전달합니다. 서브에이전트의 최종 메시지는 사용자에게 보이지 않으므로 결과를 반드시 종합에 반영합니다.

### 2단계 — 리스크 종합 (순차)

세 결과가 모이면 `risk-manager`를 실행해 핵심 리스크 3~5가지·모니터링 포인트·유동성/규모(시총·거래대금)·균형 잡힌 종합의견 초안을 받습니다. 의견이 충돌하면 근거를 비교해 조정하고, 합의되지 않은 불확실성은 함께 기재합니다.

### 3단계 — 리포트 저장

5부 구성으로 `reports/$1.md`에 저장합니다: **표지 → 재무 → 차트 → (가격예측 시나리오) → 리스크 → 종합의견**. 첫머리에 "무료 공개 데이터 기반 학습용" 한 줄, 끝에 데이터 출처·기준일 목록과 학습용 고지를 답니다.

### 4단계 — 품질 검증

`report-quality-auditor`를 실행해 수치-데이터 일치·단위·출처 누락·가드레일 위반을 점검합니다. **보류** 판정이면 지적사항을 반영해 `reports/$1.md`를 수정하고, 통과될 때까지 반복합니다.

### 5단계 — PPTX 내보내기

`report-pptx` 스킬을 호출해 `reports/$1.md` → `reports/$1.pptx`(KB 디자인, 맑은 고딕, 7슬라이드)로 변환합니다. 완료되면 산출 파일 경로(.md, .pptx)를 수석에게 보고합니다.

## 위임 원칙

- 데이터는 한 번만 수집해 공유하고, 애널리스트는 수집이 아니라 해석에 집중하게 합니다.
- 가격예측은 신호·시나리오를 산출하되 단독 결론을 그대로 채택하지 않고 다른 분야와 교차검증합니다.
- 새 분석 영역이 필요하면 임의 분석 대신 애널리스트(에이전트)를 추가합니다.
