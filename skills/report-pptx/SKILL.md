# report-pptx

`reports/{종목명}.md` 리서치를 **고정된 7장 구성의 디자인 PPTX**로 변환하는 스킬입니다.
파싱·가드레일 점검은 이 문서의 지침대로 수행하고, 디자인·렌더링은 `render.py`가 결정형으로 처리합니다.

## 입출력

- 입력: `reports/{종목명}.md` (이미 작성된 종목 리서치)
- 출력: `reports/{종목명}.pptx`
- 종목명은 사용자 인자에서 받습니다. 인자가 없으면 `reports/`의 `.md` 목록을 보여주고 어느 것을 변환할지 묻습니다.

## 슬라이드 순서 (항상 이 순서로 고정)

1. **표지** — 종목명 · 작성일(기준일) · 종합의견
2. **종목 개요** — 사업/종목 한눈 요약
3. **재무 요약** — 표 + 최근 3개년
4. **가격 / 추세** — 가격·이동평균·변동률(차트 부)
5. **뉴스 · 심리** — 핵심 이슈 + 한 줄 시장 심리
6. **리스크** — 핵심 위험요인
7. **한 줄 종합** — 종합의견 헤드라인 + 근거

## 절차

1. `reports/{종목명}.md`를 Read로 읽습니다.
2. 내용을 아래 **JSON 스키마**로 구조화합니다. 본문의 표는 그대로 `headers`/`rows`로 옮기고, 불릿은 핵심만 추립니다.
3. JSON을 `reports/.cache/{종목명}.json`에 Write로 저장합니다(폴더 없으면 생성).
4. 다음을 실행합니다(플러그인 설치 시 경로 자동 해석):
   `python "${CLAUDE_PLUGIN_ROOT}/skills/report-pptx/render.py" "reports/.cache/{종목명}.json" "reports/{종목명}.pptx"`
5. 실행 결과(`OK slides=7 ...`)를 확인하고 사용자에게 산출 경로를 보고합니다.

## 가드레일 (JSON 구성 단계에서 반드시 적용)

- **출처 없는 수치는 싣지 않습니다.** 모든 수치 텍스트 끝에 `(출처/기준일)`을 함께 넣습니다. 원문에 출처·기준일이 없는 숫자는 슬라이드에서 제외합니다.
- **매수/매도·목표가를 단정하지 않습니다.** "매수", "매도", "목표가 N원" 류 표현은 옮기지 않고, 판단 근거(추세·시나리오·조건)까지만 적습니다. 가격 예측은 범위/시나리오 형태로만 옮깁니다.
- 문체는 원문대로 "~입니다" 체를 유지합니다.
- 표는 행이 많으면 render.py가 자동으로 줄이지만, **가능하면 최근 3개년·핵심 지표 위주로 8행 이내**가 되도록 미리 추립니다.
- 자산유형/종목 성격에 맞지 않는 항목(예: 단일 종목인데 부동산 자산유형 상관)은 "해당 없음"으로 명시하거나 제외합니다.

## JSON 스키마

```json
{
  "ticker_name": "율촌화학(008730)",
  "date": "2026-06-11",
  "opinion": "중립적",
  "overview": {
    "bullets": ["사업 한 줄 요약 (출처/기준일 필요 시)", "..."],
    "table": null,
    "notes": ["보조 설명(선택)"]
  },
  "financials": {
    "table": {
      "headers": ["항목(억원)", "2023", "2024", "2025"],
      "rows": [["매출액", "4,144.9", "4,570.8", "4,853.8 (DART, 2025 연간)"]]
    },
    "bullets": ["핵심 코멘트 (출처/기준일)"],
    "notes": ["연결 기준 등 단위·기준 메모"]
  },
  "price": {
    "table": {"headers": ["항목", "값", "비고"], "rows": [["현재가", "17,110원", "FDR, 2026-06-11"]]},
    "bullets": ["추세 코멘트 (출처/기준일)"],
    "notes": ["데이터 연속성 메모 등"]
  },
  "news": {
    "bullets": ["핵심 이슈 (출처, 날짜)", "..."],
    "sentiment": "긍정적/중립적/부정적 — 근거 한 줄"
  },
  "risk": {
    "bullets": ["핵심 리스크 (출처/기준일)", "..."],
    "notes": ["데이터 한계 메모(선택)"]
  },
  "summary": {
    "headline": "한 줄 종합 (단정 금지, 방향·조건)",
    "bullets": ["근거 1", "근거 2"]
  }
}
```

- 비어있는 부(예: 표 없음)는 `null` 또는 생략하면 render.py가 건너뜁니다.
- 슬라이드당 표 8행·불릿 7~8개를 넘기지 않도록 추립니다(넘침 방지).

## 디자인 (render.py 고정)

- 포인트색 KB 옐로우 `#FFBC00`, 본문 진회색/화이트, 표 헤더 다크그레이.
- 한글 폰트 **맑은 고딕** 하나로 고정(latin/ea/cs 모두 지정 → 깨짐 방지).
- 16:9, 각 슬라이드 하단에 학습용 면책 문구 자동 삽입.
